Projects

NTU MDP Autonomous Robot
Embedded C · STM32F4 · FreeRTOS · ICM20948 · Sensor Fusion
↗
9 FreeRTOS tasks Sub-cm alignment 10×10 grid maze 180 MHz STM32F4
Problem
Navigate a 10×10 maze fully autonomously — find all targets, compute the fastest return path, execute without human input.
Approach
Wrote the entire STM32 firmware: 9 concurrent FreeRTOS tasks fusing ICM20948 IMU, dual quadrature encoders, HC-SR04 ultrasonic and Sharp IR — PID motor control + DMA-buffered UART to Raspberry Pi.
Result
Reliable navigation over repeated runs with sub-centimetre wall alignment. Gyro-corrected turns held heading error under 1° across 90° manoeuvres.
Insight
80% of autonomous reliability is firmware, not algorithm. Interrupt timing, DMA buffer sizing and task priority tuning mattered more than path planning.
Extended Kalman Filter — Sensor Fusion
Python · NumPy · State Estimation · Robotics
↗
2.04 cm RMSE +65.7% vs dead reckoning IMU + Encoders + Ultrasonic
Problem
Dead reckoning alone drifts to 6 cm error over a 27-second run. Each turn compounds heading error into position error.
Approach
EKF with arc-based differential-drive motion model, ICM20948 gyro correction at 100 Hz, and HC-SR04 updates at 20 Hz with 3σ outlier gating. All noise parameters from datasheets.
Result
2.04 cm RMSE vs 5.96 cm dead reckoning — 65.7% improvement. Uncertainty ellipses tighten visibly at each ultrasonic update.
Insight
Gyro correction at 100 Hz matters more than ultrasonic — correcting heading prevents position errors from ever accumulating.
Edge ML — 1-D CNN on Arduino BLE Sense 33
PyTorch · Edge Inference · HAR · TorchScript · 50 Hz IMU
↗
42k parameters ~165 KB float32 6-class · 50 Hz Fits 256 KB SRAM
Problem
Classify 6 activities from raw 6-axis IMU windows in real time, on a microcontroller with 256 KB RAM.
Approach
3-block 1D CNN + residual layer + Global Average Pooling. AdamW with cosine LR decay, early stopping. Exported via TorchScript for embedded deployment.
Result
Model fits comfortably after int8 quantisation (~11 KB). RF baseline hit 100% on clean data; CNN targets 96.5% on real sensor noise.
Insight
Global Average Pooling beats Flatten + dense — halves parameter count, forces conv layers to learn meaningful temporal features.
Are SHAP & LIME Faithful on Foundation Models?
TabPFN · SHAP · LIME · Explainable AI Research · NTU
↗
Infidelity 0.016 Spearman ρ = 0.47 LIME ρ = −0.06 TabPFN 90% acc
Problem
SHAP assumes fixed model weights. TabPFN uses in-context learning — when SHAP masks a feature, attention shifts silently. Are explanations still meaningful?
Approach
Measured Infidelity, Sufficiency and Spearman ρ against XGBoost-TreeSHAP (exact Shapley reference). Three explainers + an intrinsic finite-difference sensitivity probe.
Result
SHAP is locally faithful (ρ=0.47). LIME is noise (ρ=−0.06). Intrinsic probe is negatively correlated (ρ=−0.35) — the two models learned entirely different decision boundaries.
Insight
Post-hoc tools can be locally consistent and still miss the model's actual reasoning. "Faithful" only means something relative to what you're comparing against.
